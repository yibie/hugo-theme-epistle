Heti 排版功能展示

亲爱的读者，

这篇文章展示在普通书信行文中如何调用 Heti 的排版能力。Heti 的大部分增强是自动生效的，无需任何标记——中西文混排间距、标点挤压、网格对齐，都由 JS 和 CSS 在后台处理。

需要主动标记的，只有下面几类场景。

一、自动生效的能力（无需任何操作）

写作时正常书写即可，以下效果自动触发：

  • 中英混排间距：文字中夹杂English单词时，Heti 自动在中英之间插入¼字宽间距
  • 标点挤压：连续标点「，」「。」「、」自动压缩为正常视觉间距
  • 网格对齐：所有文字贴合字体基线网格排列
  • 全角引号：中文语境下自动使用「直角引号」

二、行内元信息（来源、注释）

引用他人文字时，用 meta shortcode 标注出处：

花自飘零水自流，一种相思，两处闲愁。 [宋]李清照《一剪梅》

也可以直接写原生 HTML：

这是一段引用。——出自某处

三、嵌入诗词块

信中引用诗词，用 verse shortcode 创建居中的诗词版式：

九月九日忆山东兄弟

[唐] 王维

独在异乡为异客 ，

每逢佳节倍思亲 。

遥知兄弟登高处 ，

遍插茱萸少一人 。

heti-hang 使末尾标点悬挂于行外，避免右侧视觉不齐整。

四、嵌入古文块

引用文言文段落，用 ancient shortcode：

岳阳楼记（节选）范仲淹

庆历四年春，滕子京谪守巴陵郡。越明年，政通人和，百废具兴，乃重修岳阳楼，增其旧制，刻唐贤今人诗赋于其上，属予作文以记之。

五、行间注

书信引用古文典故时，可为生僻字或专名加行间注释，注文出现在字的正上方，与注音不同——行间注写的是释义，而非读音：

臣本布衣（平民百姓），躬耕于南阳（今河南南阳），苟全性命于乱世，不求闻达于诸侯。

行内单处注释可直接写原生 <ruby> 标签，或用 ruby shortcode：「 赫蹏（古代薄纸） 」——Heti 名字即取自此典。

六、多栏排版

适合较长的列举或并列内容：

左栏内容

这里是第一栏的文字，适合两个对比主题并排展示。

右栏内容

这里是第二栏的文字，读者可以左右对照阅读。

七、跳过 Heti 处理

某些代码、外文段落不需要 Heti 干预，用 heti-skip 类：

This paragraph contains English-only content that should not be processed by Heti's CJK spacing rules.

八、Markdown 块级属性（直接附加类名）

开启 markup.goldmark.parser.attribute.block = true 后，可直接在 Markdown 块后附加类：

这是一段需要放大显示的重要话语。

引用块可以这样加元信息样式。
